## Message Flow

### Inbound Messages

<FlowSteps>
  <Step number={1} title="Webhook/Poll">
    <p>Channel receives message from platform (webhook push or polling).</p>
  </Step>
  
  <Step number={2} title="Parse">
    <p>Raw platform message is parsed into structured format.</p>
  </Step>
  
  <Step number={3} title="Normalize">
    <p>Message is converted to AeonSage unified format.</p>
  </Step>
  
  <Step number={4} title="Route">
    <p>Gateway routes message to appropriate session/handler.</p>
  </Step>
  
  <Step number={5} title="Process">
    <p>AI generates response, skills are executed.</p>
  </Step>
</FlowSteps>

### Outbound Messages

<FlowSteps>
  <Step number={1} title="Generate">
    <p>Response content generated by AI or skill.</p>
  </Step>
  
  <Step number={2} title="Format">
    <p>Content formatted for target channel capabilities.</p>
  </Step>
  
  <Step number={3} title="Chunk">
    <p>Long messages split according to channel limits.</p>
  </Step>
  
  <Step number={4} title="Send">
    <p>Message delivered via channel API.</p>
  </Step>
</FlowSteps>

## Channel Configuration

### Per-Channel Settings

Each channel supports common and specific configuration options:

<ConfigSections>
  <Section name="Common Settings">
    <Table>

          <th>Setting</th>

### Configuration Example

```json
{
  "channels": {
    "telegram": {
      "enabled": true,
      "botToken": "YOUR_BOT_TOKEN",
      "allowlist": ["123456789"],
      "prefix": "!",
      "webhookUrl": "https://your-gateway.com/webhook/telegram"
    },
    "discord": {
      "enabled": true,
      "botToken": "YOUR_BOT_TOKEN",
      "applicationId": "YOUR_APP_ID",
      "allowlist": ["123456789012345678"],
      "guildId": "YOUR_GUILD_ID"
    }
  }
}
```

## Allowlist System

The allowlist system controls who can interact with your AeonSage instance.

### Allowlist Modes

<Table>
  <thead>
    <tr>
      <th>Mode</th>
      <th>Behavior</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td><code>open</code></td>
      <td>Anyone can interact (not recommended for production)</td>
    </tr>
    <tr>
      <td><code>allowlist</code></td>
      <td>Only listed users/groups can interact</td>
    </tr>
    <tr>
      <td><code>admin</code></td>
      <td>Only admin users can interact</td>
    </tr>
  </tbody>
</Table>

### Managing Allowlists

```bash
# Add user to allowlist
aeonsage channels allow add telegram user:123456789

# Add group to allowlist
aeonsage channels allow add discord group:987654321

# List allowlist entries
aeonsage channels allow list telegram

# Remove from allowlist
aeonsage channels allow remove telegram user:123456789
```

## Troubleshooting

### Common Channel Issues

<TroubleshootList>
  <Issue name="Channel not receiving messages">
    <p>Verify webhook URL is accessible from internet.</p>
    <p>Check channel credentials are correct.</p>
    <p>Ensure channel is enabled in configuration.</p>
  </Issue>
  
  <Issue name="Messages not sending">
    <p>Check API rate limits for the channel.</p>
    <p>Verify bot has necessary permissions.</p>
    <p>Check gateway logs for error details.</p>
  </Issue>
  
  <Issue name="Allowlist blocking authorized users">
    <p>Verify user ID format matches channel expectations.</p>
    <p>Check if group vs user allowlist is correct.</p>
    <p>Use <code>aeonsage channels status --probe</code> to debug.</p>
  </Issue>
</TroubleshootList>

### Debug Commands

```bash
# Check channel status
aeonsage channels status

# Probe channel connectivity
aeonsage channels status --probe

# View channel logs
aeonsage logs --channel telegram

# Test message sending
aeonsage message send telegram:@user "Test message"
```
