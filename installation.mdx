## CLI Installation

Install the AeonSage CLI for command-line operation and Gateway server.

### Using pnpm (Recommended)

```bash
pnpm add -g aeonsage
```

### Using npm

```bash
npm install -g aeonsage
```

### Using yarn

```bash
yarn global add aeonsage
```

### Verify Installation

```bash
aeonsage --version
# aeonsage/2026.2.15

aeonsage --help
```

### Quick Start with CLI

```bash
# Initialize configuration
aeonsage init

# Start the Gateway
aeonsage gateway start

# Open in browser (default port)
open http://localhost:3000
```

## Docker Installation

Run AeonSage in a container for isolated, reproducible deployments.

### Pull Image

```bash
docker pull aeonsage/gateway:latest
```

### Quick Run

```bash
docker run -d \
  --name aeonsage-gateway \
  -p 18789:18789 \
  -v aeonsage-data:/root/.aeonsage \
  -e GATEWAY_TOKEN=your-secure-token \
  -e OPENAI_API_KEY=your-key \
  aeonsage/gateway:latest
```

### Docker Compose

Create `docker-compose.yml`:

```yaml
version: '3.8'

services:
  aeonsage:
    image: aeonsage/gateway:latest
    container_name: aeonsage-gateway
    restart: unless-stopped
    ports:
      - "18789:18789"
    volumes:
      - ./config:/root/.aeonsage
      - ./skills:/root/.aeonsage/skills
    environment:
      - GATEWAY_TOKEN=${GATEWAY_TOKEN}
      - OPENAI_API_KEY=${OPENAI_API_KEY}
      - ANTHROPIC_API_KEY=${ANTHROPIC_API_KEY}
      - LOG_LEVEL=info
```

Start:

```bash
docker-compose up -d
```

## Build from Source

For development or custom builds.

### Prerequisites

- **Node.js 22+**
- **pnpm** (recommended) or npm
- **Git**

### Build Steps

```bash
# Clone the repository
git clone https://github.com/aeonsage/aeonsage.git
cd aeonsage

# Install dependencies
pnpm install

# Build the project
pnpm build

# Run locally
pnpm aeonsage gateway start
```

### Development Mode

```bash
# Run in development mode with hot reload
pnpm dev
```

## Platform-Specific Notes

### macOS

<Accordion>
  <AccordionItem title="Apple Silicon vs Intel">
    
    Determine your Mac's architecture:
    
    ```bash
    uname -m
    # arm64 = Apple Silicon (M1/M2/M3)
    # x86_64 = Intel
    ```
    
    Download the matching DMG file.
    
  </AccordionItem>
  
  <AccordionItem title="Permissions">
    
    macOS may request permissions for:
    
    - **Accessibility**: For keyboard shortcuts
    - **Screen Recording**: For screenshot tools
    - **Full Disk Access**: For file operations
    
    Grant only permissions you need.
    
  </AccordionItem>
</Accordion>

### Windows

<Accordion>
  <AccordionItem title="Antivirus Exclusion">
    
    Some antivirus software may flag AeonSage. Add an exclusion if needed:
    
    1. Open Windows Security
    2. Go to **Virus & threat protection â†’ Manage settings**
    3. Add exclusion for the AeonSage folder
    
  </AccordionItem>
  
  <AccordionItem title="WSL2 Integration">
    
    AeonSage runs on Windows natively. For WSL2:
    
    ```bash
    # In WSL2
    npm install -g aeonsage
    aeonsage gateway start --host 0.0.0.0
    
    # Access from Windows (check actual port in Gateway logs)
    # http://localhost:PORT
    ```
    
  </AccordionItem>
</Accordion>

### Linux

<Accordion>
  <AccordionItem title="System Dependencies">
    
    For Debian/Ubuntu:
    
    ```bash
    sudo apt-get update
    sudo apt-get install -y libwebkit2gtk-4.1-dev \
      build-essential \
      curl \
      wget \
      libssl-dev \
      libgtk-3-dev \
      libayatana-appindicator3-dev \
      librsvg2-dev
    ```
    
  </AccordionItem>
  
  <AccordionItem title="Headless Server">
    
    For servers without GUI:
    
    ```bash
    # Install CLI only
    npm install -g aeonsage
    
    # Start in background
    aeonsage gateway start --daemon
    ```
    
  </AccordionItem>
</Accordion>
