## Allowlist System

### Access Control Modes

<AccessControlTable>
  <thead>
    <tr>
      <th>Mode</th>
      <th>Behavior</th>
      <th>Use Case</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td><code>open</code></td>
      <td>Anyone can interact</td>
      <td>Public demos, testing</td>
    </tr>
    <tr>
      <td><code>allowlist</code></td>
      <td>Only listed users/groups</td>
      <td>Private deployments</td>
    </tr>
    <tr>
      <td><code>admin</code></td>
      <td>Only admin users</td>
      <td>Secure environments</td>
    </tr>
  </tbody>
</AccessControlTable>

### Allowlist Management

```bash
# Add user to allowlist
aeonsage channels allow add telegram user:123456789

# Add group to allowlist
aeonsage channels allow add discord group:987654321

# List allowlist entries
aeonsage channels allow list telegram

# Remove from allowlist
aeonsage channels allow remove telegram user:123456789
```

## Troubleshooting

### Common Issues and Solutions

<TroubleshootAccordion>
  <Issue name="Channel not receiving messages">
    <DiagnosticSteps>
      <Step>Verify webhook URL is accessible from internet</Step>
      <Step>Check channel credentials are correct</Step>
      <Step>Ensure channel is enabled in configuration</Step>
      <Step>Review gateway logs for errors</Step>
    </DiagnosticSteps>
    <Solution>
      <code>aeonsage channels status --probe</code>
    </Solution>
  </Issue>
  
  <Issue name="Messages not sending">
    <DiagnosticSteps>
      <Step>Check API rate limits for the channel</Step>
      <Step>Verify bot has necessary permissions</Step>
      <Step>Check gateway logs for error details</Step>
    </DiagnosticSteps>
  </Issue>
  
  <Issue name="Allowlist blocking authorized users">
    <DiagnosticSteps>
      <Step>Verify user ID format matches channel expectations</Step>
      <Step>Check if group vs user allowlist is correct</Step>
      <Step>Use debug command to test</Step>
    </DiagnosticSteps>
    <Solution>
      <code>aeonsage channels allow list telegram --verbose</code>
    </Solution>
  </Issue>
</TroubleshootAccordion>
